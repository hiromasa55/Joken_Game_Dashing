# Joken_Game_Dashing

就職活動用に制作した、2D横スクロールアクションゲームの協力開発プロジェクトです。
Unityの教材をベースに、チーム独自のオリジナル要素を多数追加して発展させました。

## 🎮 ゲーム概要
プレイヤーを操作し、迫り来る無敵の敵キャラクターを避けながら、各ステージのゴールを目指すアクションゲームです。

### 🌟 オリジナル追加要素
ベースとなる教材から、以下の機能を独自に拡張・実装しました。
* **4面構成のステージ選択**: プレイヤーが自由に遊ぶステージを選択できるメニュー機能を実装。
* **追尾型無敵キャラクター**: 操作キャラクターを執拗に追尾してくる、緊張感を高める敵を配置。
* **ゴール判定とクリア機能**: 特定のゴールオブジェクトに触れることで、ステージクリアとなるシーケンスを追加。
* **コントローラー対応**: キーボード操作だけでなく、ゲームパッドでの快適なプレイに対応。
* **各種サウンド（BGM/SE）の追加**: 臨場感を高める音響効果の実装。
* **オリジナルテクスチャ＆アニメーション**: 素材サイトに頼らず、独自の世界観を表現。

## 👥 開発体制と私の役割（チームリーダー）

### 🗓️ 開発期間
* **3週間**（企画・素材制作・実装・統合）

### 👥 チーム構成（4名）
初めての共同開発を円滑に進めるため、リーダーとして開発手法を工夫し、役割分担を行いました。

* **チームリーダー（私）**:
    * **システム設計・プログラミング**: ゲームの根幹となるすべてのスクリプト（プログラム）の設計・記述を担当。
    * **開発環境の整備**: 各メンバーが迷わず作業できるよう、機能ごとに「Prefab」としてコンポーネント化し、配布する仕組みを構築。
* **メンバー（3名）**:
    * **レベルデザイン**: 各自1ステージずつ担当し、計4ステージの構成・調整。
    * **アセット制作**: 担当ステージに使用するオリジナルテクスチャの作成およびアニメーション設定。

## 🚀 本プロジェクトにおける課題とリーダーとしての学び

本プロジェクトはチーム全員にとって**初めてのGit/GitHubを用いた共同開発**でした。リーダーとして技術的な主導権を持ちつつ、チーム開発のプロセスにおいて多くの気づきを得ました。

### 1. 効率的なアセット管理とマネジメント（成果）
* **取り組み**: 短い制作期間（3週間）のなかで、メンバーの作業が競合しないよう、私が共通のスクリプトを記述し、メンバーにはそれを「Prefab」として配置・拡張してもらう手法を採りました。
* **結果**: プログラムのコンフリクト（衝突）を最小限に抑えつつ、初学者のメンバーでもスムーズにステージ作成に集中できる環境を作ることができ、期間内に4面構成のゲームを完成させました。オブジェクト指向の思想を活かした設計の重要性を実感しました。

### 2. Git/GitHubの活用における課題
* **課題**: チーム全員がGit管理に不慣れだったため、Sceneファイルのコンフリクト解消やブランチの運用サポートに追われ、Issue や Pull Request 機能を十分に活用したタスク管理・コードレビューまで手が回りませんでした。
* **学び・今後の対策**: 複数人でUnityプロジェクトを触る際のコンフリクト防止の難しさを痛感しました。この経験から、次回の開発では「作業範囲の厳密な切り分け（Scene分割など）」を事前に行い、Git Flowに則ったブランチ運用、Issueによるタスクの可視化を徹底することで、より強固なチームマネジメントを行いたいと考えています。
